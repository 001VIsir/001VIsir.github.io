Spring Boot 自动配置原理

前言

Spring Boot 最大的特点就是”约定大于配置”，我们只需要很少的配置就能启动一个应用。这背后的核心就是自动配置机制。

@SpringBootApplication 的秘密

@SpringBootApplication 是一个组合注解，拆解开来：

核心是 @EnableAutoConfiguration，它导入了 AutoConfigurationImportSelector。

自动配置的核心流程

1. Spring Boot 会扫描什么？

启动时会扫描所有 jar 包中的：

  • META-INF/spring.factories
  • META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports

这两个文件里定义了需要自动配置的类。

2. @Conditional 条件注解

不是所有配置都会生效，每个自动配置类都带了一系列条件注解：

常见条件注解：

注解 作用
@ConditionalOnClass classpath 有指定类才生效
@ConditionalOnMissingBean 容器中没有指定 Bean 才生效
@ConditionalOnProperty 配置文件中指定属性才生效
@ConditionalOnWebApplication 是 Web 应用才生效

3. 配置属性绑定

自动配置类会通过 @EnableConfigurationProperties 绑定配置文件中的属性：

Spring Boot 会把这些配置绑定到 RedisProperties 对象上，自动注入到自动配置类中使用。

如何自定义自动配置

可以自己写一个 starter，实现类似官方 starter 的效果：

  1. 创建一个 jar，包含自动配置类和业务类
  2. 在 META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports 中注册
  3. 其他项目引入这个 jar 的依赖即可自动生效

总结

Spring Boot 的自动配置通过 @EnableAutoConfiguration 触发，扫描 spring.factories 和 AutoConfiguration.imports 文件，根据各种 @Conditional 条件注解判断是否生效，最后绑定配置文件中的属性。整个过程无需人工干预，大大简化了开发配置。
