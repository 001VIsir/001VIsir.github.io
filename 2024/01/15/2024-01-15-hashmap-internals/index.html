HashMap 底层原理详解

前言

HashMap 是 Java 开发中最常用的集合之一，也是面试中的高频考点。理解它的底层原理，对于写出高质量的代码至关重要。

数据结构

JDK 1.8 之前，HashMap 底层是数组 + 链表的组合。JDK 1.8 做了优化，引入红黑树，当链表长度超过 8 且数组长度大于 64 时，链表会转换成红黑树。

核心参数

  • capacity — 数组长度，默认 16
  • loadFactor — 负载因子，默认 0.75
  • threshold — 扩容阈值，等于 capacity × loadFactor

当元素数量超过 threshold 时，HashMap 会自动扩容为原来的 2 倍。

put 过程

为什么 HashMap 不是线程安全的？

  1. 扩容时链表可能死循环 — 多个线程同时触发扩容，1.7 中会导致链表环形引用
  2. put 时数据覆盖 — 并发 put 可能丢失数据
  3. size 不准确 — 并发修改计数器可能出错

线程安全的替代方案：ConcurrentHashMap、Hashtable、Collections.synchronizedMap()

总结

HashMap 通过哈希算法将 key 映射到数组下标，结合链表或红黑树解决哈希冲突。理解它的底层数据结构和工作流程，是掌握 Java 集合框架的重要基础。
