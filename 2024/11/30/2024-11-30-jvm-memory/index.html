JVM 内存区域划分详解

前言

JVM（Java Virtual Machine）是 Java 程序运行的基础环境。理解 JVM 的内存区域划分，是排查 OOM（OutOfMemoryError）和进行性能调优的前提。

运行时数据区域

JVM 在执行 Java 程序时，会把内存划分为以下几个区域：

1. 程序计数器

一小块内存，当前线程执行的字节码行号指示器。如果执行 Native 方法，则为空。

特点：

  • 线程私有，每个线程都有一个
  • 唯一不会发生 OOM 的区域

2. 虚拟机栈

每个线程调用方法时会在栈中创建一个栈帧（Stack Frame），方法执行完毕后栈帧出栈。

栈帧中包含：

  • 局部变量表 — 方法参数和局部变量
  • 操作数栈 — 中间计算结果
  • 动态链接 — 指向运行时常量池的引用
  • 返回地址 — 方法退出地址

常见异常：

  • StackOverflowError — 线程请求的栈深度超过最大值（递归调用没有正确退出）
  • OutOfMemoryError — 动态扩展栈时无法申请足够内存

3. 本地方法栈

与虚拟机栈类似，不过是为 Native 方法服务的。一些 JVM（如 HotSpot）把两者合并了。

4. 堆（Heap）

最重要的区域，几乎所有对象实例和数组都在堆上分配。

默认比例（CMS / Parallel Old）：Eden : S0 : S1 = 8 : 1 : 1

5. 方法区（Method Area）

存储类信息（类的结构、字段、方法）、常量、静态变量、JIT 编译后的代码。

JDK 8 之前用永久代（PermGen）实现，JDK 8 之后改为元空间（Metaspace），使用直接内存，不在 JVM 堆内。

常见 OOM：MetaspaceSize 不足会抛 OOM。

总结

JVM 内存分为线程共享（堆、方法区）和线程私有（栈、程序计数器）两大类。堆是 OOM 的重灾区，栈溢出多由递归调用引发。理解每个区域的作用和存储内容，是进行 JVM 调优和问题排查的基础。
