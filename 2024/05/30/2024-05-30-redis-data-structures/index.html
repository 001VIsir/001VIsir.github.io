Redis 5 种数据结构详解

前言

Redis 之所以快，是因为它把数据都放在内存里。但更重要的是它提供了丰富的数据结构，能高效地解决各种业务问题。

1. String（字符串）

最基本的数据类型，Memcached 也只支持这一种。

常用命令

应用场景

  • 缓存简单的序列化的 JSON 对象
  • 分布式 session
  • 计数器
  • 分布式锁（SETNX + EXPIRE）

底层实现

Redis 3.2 之前 SDS（Simple Dynamic String）有多种结构。3.2 之后统一使用 robj 包装，字符串值本身是 sdshdr，包含长度、可用空间等元数据，比 C 字符串更安全且支持动态扩容。

2. Hash（哈希）

类似 Java 的 HashMap，适合存储对象。

常用命令

应用场景

  • 存储用户对象信息（比 JSON 字符串更节省修改单个字段的资源）
  • 购物车商品

3. List（列表）

按插入顺序排序的列表，底层是双向链表。

常用命令

应用场景

  • 消息队列（但生产环境推荐用 Stream）
  • 最新文章列表
  • 评论列表

4. Set（集合）

无序不重复的元素集合，支持交、并、差运算。

常用命令

应用场景

  • 标签系统（共同关注、共同好友）
  • UV 统计（去重计数）
  • 黑名单/白名单

5. Sorted Set（有序集合）

每个元素关联一个分数，按分数排序。

常用命令

应用场景

  • 排行榜（积分排序）
  • 延时队列
  • 热榜排序

底层实现

当元素数量少时使用ziplist（压缩列表），元素多了自动转换为 skiplist（跳表）+ dict（哈希表）的组合。跳表支持快速范围查询，哈希表支持 O(1) 查找分数。

选择建议

场景 推荐类型
缓存序列化对象 String
存储用户属性 Hash
最新消息队列 List
标签/去重 Set
排行榜 Sorted Set
