Spring Boot Starter 原理

前言

我们用 Spring Boot 时，只要引入一个 starter 依赖，就能直接使用其中的功能，比如引入 spring-boot-starter-web 就能开发 Web 项目。这个机制是怎么工作的？

什么是 Starter？

Starter 就是一个包含两部分内容的 jar 包：

  1. 自动配置模块（xxx-autoconfigure）— 包含自动配置类和所有需要用到的依赖
  2. Starter 模块（xxx-starter）— 只是一个空 jar，依赖自动配置模块，用于给其他项目引用

总结

Starter 的核心是自动配置：Spring Boot 根据 classpath 中的类、配置的 properties、Spring 容器中的 Bean 等条件，自动决定哪些配置生效。starter 依赖把需要的 jar 都打包进来，配置属性通过 @EnableConfigurationProperties 注入，做到了”零配置”或”少配置”就能运行一个完整功能。
