java序列化与反序列化

前言

java对象序列化与反序列化

序列化

序列化（Serialization）：将 Java 对象转换为字节序列的过程（对象 → 字节）

java原生序列化

  1. 实现 Serializable 接口
  1. 序列化到文件

反序列化

特性 数组 ArrayList LinkedList
初始化机会 固定 动态扩展 动态扩展
存储方式 连续内存 连续内存（Object[]） 链表节点（离散内存）
随机访问 1 1 n
头部删除/插入 n n 1
尾部插入/删除 1 1 1
中间插入/删除 n n n
内存占用 低 Object开销 每个节点多两个引用

核心区别

典型应用场景

场景 说明
固定大小、高频随机访问 数组
需要动态增长、高频随机访问 ArrayList
高频头尾插入删除 LinkedList
中间插入删除为主 ArrayList

核心机制

ArrayList：基于 Object[]，扩容时创建新数组（1.5倍），复制元素

LinkedList：基于双向链表（Node prev/item/next），无容量概念

性能关键：ArrayList 的连续内存带来更好的 CPU 缓存命中率，实际通常比 LinkedList 快。

反射

反射是 Java 在运行时动态获取类的信息、操作对象成员的能力。通过java.lang.reflect包实现。

核心类

类 作用
Class 类的元数据（类名、构造器、字段、方法）
Field 字段（属性）
Method 方法
Constructor 构造器

获取Class对象

作用

  1. 获取类的成员
  1. 动态创建对象
  1. 动态调用方法
  1. 动态访问/修改字段
  1. 获取泛型信息

应用场景

  1. 框架（Spring / Hibernate）
  1. 序列化/反序列化（JSON / Jackson）
  1. 动态代理
  1. 依赖注入容器
  1. 注解处理器
  1. 测试框架（JUnit）

优缺点

优点 缺点
运行时动态能力，极大灵活性 性能开销大（比直接调用慢 10-100 倍）
解耦：代码不依赖具体类 破坏封装性（可访问私有成员）
支持框架和插件机制 安全限制（模块化系统）
动态代理、AOP 基础 可读性差，难以调试
