Redis 如何实现分布式锁

前言

在单机环境下，我们可以用 synchronized 或 ReentrantLock 控制并发。但在分布式环境下，不同机器上的进程需要协调，就必须用分布式锁。Redis 是实现分布式锁最常用的方案之一。

分布式锁的核心要求

  1. 互斥性 — 同一时刻只能有一个客户端持有锁
  2. 防死锁 — 客户端崩溃时，锁能自动释放（TTL）
  3. 可重入 — 同一客户端可以多次获取锁
  4. 高性能 — 加锁解锁要快

最简单的实现

NX 保证互斥，EX 设置过期时间防止死锁。

安全解锁：校验持有者

上面的解锁直接删除 key 有问题——如果 key 过期后其他客户端获取了锁，你的客户端再删除就会误删别人的锁。

正确做法：删除前先检查 value 是否是自己设置的。

Redis 分布式锁的缺陷

Redis 分布式锁在主从切换时存在缺陷：如果 Master 宕机，从库晋升但锁信息还没同步，可能导致两个客户端同时持有锁。

解决方案：

  • 使用 RedLock（多个独立 Redis 实例加锁，需要 N/2+1 以上成功才算成功）
  • 使用 etcd 或 ZooKeeper 实现分布式锁

总结

Redis 分布式锁的核心是 SET NX EX 原子命令，解锁要用 Lua 脚本保证原子性。生产环境推荐使用 Redisson，避免重复造轮子。如果对一致性要求极高，考虑使用 etcd 或 Zookeeper。
