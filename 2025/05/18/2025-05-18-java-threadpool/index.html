Java 线程池参数和拒绝策略

前言

线程的创建和销毁需要系统调用，开销很大。在高并发场景下，频繁创建销毁线程会严重影响性能。线程池通过复用线程来避免这个问题，是 Java 并发编程的核心组件。

线程池的七大参数

参数详解

corePoolSize — 线程池中始终保持的线程数量（即使它们空闲）。核心线程不会因空闲时间而销毁。

maximumPoolSize — 线程池允许的最大线程数。当任务队列满了，会创建非核心线程来处理新任务。

keepAliveTime — 非核心线程空闲超过这个时间，就会被回收。

workQueue — 任务队列，用于存放等待执行的任务。常用实现：

  • LinkedBlockingQueue — 无界队列，容易 OOM
  • ArrayBlockingQueue — 有界队列，推荐
  • SynchronousQueue — 不存储任务，直接提交给线程

threadFactory — 创建线程的工厂，可以自定义线程名称、优先级等。

handler — 拒绝策略，当队列满了且线程数达到最大值时的处理策略。

线程池执行流程

四种拒绝策略

策略 行为
AbortPolicy（默认） 抛出 RejectedExecutionException，拒绝任务
CallerRunsPolicy 由调用者线程执行任务（提交任务的线程自己跑）
DiscardPolicy 直接丢弃任务，不抛异常
DiscardOldestPolicy 丢弃队列中最老的任务，然后尝试重新提交新任务

如何合理设置线程数

  • CPU 密集型（计算密集，如加密、解压）— 线程数 = CPU 核心数 + 1
  • IO 密集型（如网络请求、数据库查询）— 线程数 = CPU 核心数 × 2，或用公式：线程数 = CPU 核心数 × (1 + IO时间/CPU时间)

总结

线程池的核心是七大参数：核心线程数、最大线程数、空闲存活时间、存活时间单位、任务队列、线程工厂、拒绝策略。执行流程遵循”核心线程 → 队列 → 最大线程 → 拒绝策略”的顺序。生产环境应避免使用 Executors，手动创建才能精确控制参数，避免 OOM 风险。
