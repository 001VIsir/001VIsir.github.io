java多线程

前言

java多线程

线程 vs 进程

线程 vs 进程:

  • 进程：独立运行的程序（如打开两个 IDE）
  • 线程：进程内的执行单元（一个进程可有多个线程）

Java 线程：

  • main 线程：程序入口
  • JVM 自动创建多个 GC 线程

java创建线程方式

  1. 集成 Thread
  1. 实现 Runnable（这似乎最好）
  1. 实现Callable
  1. 线程池

线程生命周期

六种状态详解

状态 说明 触发方式
NEW 线程创建但未启动 new Thread()
RUNNABLE 正在jvm或者等待CPU start()
BLOCKED 等待获取锁 进入synchronized代码块
WATING 无限期等待 wait()/join()/park()
TIMED_WAITING 有限期等待 sleep(ms) / wait(ms) / join(ms)
TERMINATED 线程执行完毕 run()正常返回或抛出异常

线程同步

问题：线程不共享

问题分析：count++ 实际是 3 步操作：读取 → 加1 → 写入

解决方式：

  1. synchronized(悲观锁)

  2. Lock（手动锁）

  3. volatile（可见性）

  4. Atomic原子类（无锁）

    常用原子类：AtomicInteger、AtomicLong、AtomicBoolean、AtomicReference

生产者-消费者模型

wait/notify实现

BlockingQueue实现（推荐）

线程池

常用线程池类型

TreadPoolExecutor（自定义）

run() vs start()

核心区别

流程图

调用 start() 时：

主线程                         新线程
  │                              │
  │ new Thread()                 │
  │ ─────────────────────────────→│
  │                              │ Thread 对象创建
  │                              │
  │ start()                      │
  │ ──→ [JVM 创建新线程] ──────→  │
  │         ↓                     │
  │   Thread 状态: NEW → RUNNABLE │
  │                              │
  │  主线程继续执行               │  新线程执行 run()
  │  (不等新线程)                 │
  │                              │ t.run() 被调用
  │                              │ 输出 "run() 执行了"
  │                              │
  │                              │ run() 结束
  │                              │ 状态: RUNNABLE → TERMINATED

关键区别总结

特性 run() start()
调用方式 普通方法调用 启动线程
执行线程 当前线程 新线程
调用次数 可多次调用 只能调用一次否则 (IllegalThreadStateException）
返回值 无 void
是否异步 否 是

总结

创建线程:
├── extends Thread
├── implements Runnable（常用）
├── implements Callable + FutureTask（有返回值）
└── 线程池 ExecutorService（推荐）

同步机制:
├── synchronized（悲观锁）
├── Lock（手动锁）
├── volatile（可见性）
├── Atomic原子类（无锁）
└── BlockingQueue（阻塞队列）

线程池:
├── Executors 工厂方法（快速创建）
└── ThreadPoolExecutor（自定义）

异步编程:
└── CompletableFuture（链式调用）

线程安全集合:
├── CopyOnWriteArrayList
├── ConcurrentHashMap
└── BlockingQueue
