elasticsearch相关

前言

Elasticsearch 是一个基于 Apache Lucene 的分布式搜索和分析引擎，是 ELK : （Elasticsearch、Logstash、Kibana）的核心组件。

核心概念

概念 说明
Index（索引） 类似数据库，包含多篇文档
Document（文档） 类似数据库中的一行记录，JSON格式
Shard（分片） 索引的水平拆分单元，可分布式存储
Replica（副本） 分片的副本，保证高可用和读取吞吐

数据结构

  • 倒排索引：核心存储结构，term → doc 映射
  • Lucene：底层搜索引擎库，负责索引和检索

主要特点

  1. 实时搜索：新增数据秒级可查（NRT ~1s）
  2. 分布式：天然水平扩展，支持 PB 级数据
  3. 全文检索：支持分词、模糊匹配、短语查询
  4. 聚合分析：Facet、Aggregation（求和、平均、百分比等）
  5. 高可用：副本分片自动 failover
  6. RESTful API：JSON over HTTP，跨语言友好

典型应用场景

场景 说明
全文搜索 电商搜索、企业文档搜索
日志分析 ELK日志采集分析
应用性能监控 APM
安全分析 SIEM、威胁检测
地理位置搜索 附近地点查询

核心机制

Elasticsearch（基于 Apache Lucene）通过以下核心机制实现极快的查询速度：

  1. 倒排索引（Inverted Index）

这是最核心的结构。与正排索引（doc → terms）相反，倒排索引按 term 存储对应的
doc 列表：

term “apple” → [doc1, doc3, doc7]
term “banana” → [doc2, doc5]

查询时直接通过 term 找到文档，无需扫描全量数据，时间复杂度从 O(n) 降到 O(1) 或
O(log n)。

  1. 段（Segment）架构
  • 索引数据被分成多个 immutable 的段
  • 写入时先到内存 buffer，定期刷到磁盘生成新段
  • 段一旦生成就不修改（只读），避免了随机写入
  • 后台定期合并小段为大段，减少段数量
  1. 多种缓存机制
缓存类型 作用
Node Query Cache 存查询结果（按节点）
Shard Request Cache 缓存分片级别查询结果
Fielddata 聚合/排序的列式存储
Doc Values 磁盘上的列式存储，比 Fielddata 节省内存
系统页缓存（Page Cache） OS 层 MMap 加速磁盘读写
  1. 分布式并行搜索
  • 索引分为主副分片，查询可并行在所有分片上执行
  • 协调节点（Coordinating Node）汇总各分片结果
  • 副本分片同时提供读写分离，提升吞吐
  1. 向量化与跳过式扫描
  • Lucene 使用 Block Max WAND 等算法跳过不相关文档
  • 支持布尔查询、短语查询、范围查询等多种优化路径
  • 数值类型和日期类型使用 BKD 树加速范围查询
  1. Near Real-Time (NRT) 查询

新增文档写入内存 buffer 和 transact log，不立即刷盘，通过 refresh interval（约1s）生成新段即可见，实现秒级近实时搜索。

简而言之：倒排索引 + 不可变段 + 多级缓存 + 并行分片搜索 + 智能算法优化 = ES的极速查询能力。

生态组件

Logstash → 数据采集、ETL
Beats → 轻量级数据采集
Elasticsearch → 存储、搜索、分析
Kibana → 可视化

架构示意

客户端请求
↓
协调节点（Coordinating Node）
↓
数据节点（Data Node）
├── 分片 1 (Primary) → 副本 1 (Replica)
├── 分片 2 (Primary) → 副本 2 (Replica)
└── 分片 3 (Primary) → 副本 3 (Replica)

简单来说，Elasticsearch就是为大规模数据搜索和分析而生的分布式数据库，核心优势在于倒排索引 + 分布式架构带来的高速检索能力。
